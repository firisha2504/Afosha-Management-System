# 💰 Bulk Weekly Payment Recording Guide

## 🎯 What is Bulk Payment?

**Bulk Payment** allows you to record payments for **multiple members at once** instead of recording them one by one. This is perfect for Saturday meetings when collecting contributions from many members.

---

## 📍 Where to Find It?

### Location: **Payments Page**

1. Click **"Payments"** in the left sidebar
2. Look for **"Bulk Record"** button at the top right (blue button with 👥 icon)
3. Click it to open bulk payment modal

**Note**: The button is next to "Record Payment (Special/Late)" button

---

## ✅ Step-by-Step: How to Record Bulk Payments

### **Step 1: Open Bulk Payment Modal**

1. Go to **Payments** page
2. Click **"Bulk Record"** button (or 👥 Users icon)
3. Bulk payment modal opens

---

### **Step 2: Select Week/Obligation**

**You have 3 options:**

#### Option A: **Select Specific Week** (Recommended for Saturday meetings)
```
1. In "For which obligation?" dropdown
2. Select: "Week 25, 2026 - 6/27/2026"
3. This loads all members who owe for that week
```

#### Option B: **Select Special Contribution**
```
1. In "For which obligation?" dropdown
2. Select: "Special Contribution - [Campaign Name]"
3. This loads all members who owe that special contribution
```

#### Option C: **Leave Empty for Manual Entry**
```
1. Leave "For which obligation?" empty
2. Manually set amount for each member
```

---

### **Step 3: Select Members**

After selecting a week, you'll see a list of all members with checkboxes:

```
☑ ABCD (AF20263488)                    50 ETB
☑ Cala Ahmed (AF20261069)              50 ETB  
☑ Firomsa Abdi (AF20269679)            50 ETB
```

**Check the boxes** for members who paid:
- ✅ Check = Member paid
- ☐ Uncheck = Member didn't pay (skip them)

---

### **Step 4: Set Payment Amount**

You have 2 options:

#### **Option A: Same Amount for All** (Most Common)
```
1. "Amount for all" field: Enter 50
2. All checked members will be recorded with 50 ETB
```

#### **Option B: Individual Amounts** (For Partial Payments)
```
1. Leave "Amount for all" empty
2. Set individual amount for each member:
   - ABCD: 50 ETB (full)
   - Cala: 30 ETB (partial)
   - Firomsa: 50 ETB (full)
```

---

### **Step 5: Set Payment Details**

```
Payment Method: [Select one]
  - Cash (most common)
  - Bank Transfer
  - Mobile Money

Payment Date: [Today's date - auto-filled]

Notes (optional): "Saturday meeting collection"
```

---

### **Step 6: Review and Submit**

1. **Review the summary**:
   ```
   Selected: 3 members
   Total Amount: 150 ETB
   ```

2. **Click "Record 3 Payments"** button

3. **Wait for confirmation**:
   ```
   ✅ 3 payment(s) recorded successfully!
   ```

4. **Done!** All payments are now recorded

## 📊 Real-World Example: Saturday Meeting

### Scenario:
**Saturday, June 27, 2026 - Week 25 Meeting**
- 10 members attend
- 8 members pay full amount (50 ETB each)
- 2 members pay partial (30 ETB each)

### Steps:

#### **1. Open Bulk Record**
- Click "Payments" → "Bulk Record"

#### **2. Select Week 25**
- Dropdown: Select "Week 25, 2026"
- System loads all 10 members with 50 ETB each

#### **3. Select Who Paid (8 members full)**
```
☑ Member 1    50 ETB
☑ Member 2    50 ETB
☑ Member 3    50 ETB
☑ Member 4    50 ETB
☑ Member 5    50 ETB
☑ Member 6    50 ETB
☑ Member 7    50 ETB
☑ Member 8    50 ETB
☐ Member 9    (didn't pay - uncheck)
☐ Member 10   (didn't pay - uncheck)
```

#### **4. Set Amount**
- "Amount for all": **50**
- This sets 50 ETB for all 8 checked members

#### **5. Record Partial Payments Separately**
- After submitting the 8 full payments
- Open bulk record again
- Check Member 9 and Member 10
- Set individual amounts: 30 ETB each
- Submit

#### **6. Summary**
```
Full Payments: 8 × 50 ETB = 400 ETB
Partial Payments: 2 × 30 ETB = 60 ETB
Total Collected: 460 ETB
```

**Saturday meeting - all members pay 50 ETB:**

```

2. Check all members who paid (e.g., 15 members)
```

**Some pay full, some pay partial:**

**Option A: Do in 2 Batches**
```

4. Submit
```

## ⚡ Tips for Fast Recording

### **1. Prepare Before Opening Modal**
- Know which members paid
- Have list ready (paper or mental note)
- Know amounts (full or partial)

### **2. Use Same Amount When Possible**
- Most Saturday meetings: everyone pays 50 ETB
- Use "Amount for all" → Super fast!

### **3. Batch by Payment Method**
```
Batch 1: All Cash payments
Batch 2: All Bank Transfers
Batch 3: All Mobile Money
```

### **4. Record Partial Payments Separately**
- First: Record all full payments (50 ETB)
- Then: Record partial payments individually

## ⚠️ Important Notes

### **1. Status Will Be PENDING**
- All bulk payments start as PENDING
- Admin must verify them later
- Go to Payments page → Verify each

### **2. Members Not Selected = Not Recorded**
- Only checked members get payments recorded
- Double-check your selection before submit

### **3. Can't Mix Weeks**
- One bulk operation = One week
- To record multiple weeks: do separate bulk operations

### **4. Partial Payments Still Work**
- Member who paid 30 ETB today → Record 30
- Next week pays 20 ETB → Record another 20
- System adds them: 30 + 20 = 50 → PAID

## 🎓 Common Questions

### **Q: Can I record payments for different weeks in one bulk?**
A: No. One bulk operation = One week. Do separate bulks for different weeks.

### **Q: What if member paid partial amount?**
A: Either:
- Set individual amount for that member, OR
- Record full payments in bulk, then record partial separately

### **Q: Do I need to verify bulk payments?**
A: Yes, bulk payments are created as PENDING. Admin verifies them later.

### **Q: Can I edit after bulk submit?**
A: Yes! Go to Payments page, find the payment, click Edit.

### **Q: What if I made a mistake?**
A: You can:
- Edit the payment (change amount, method, etc.)
- Delete the payment (if not verified yet)
- Rollback (if already verified)
